CARLA DEEP DRIVING - RESEARCH NOTES & SIGNIFICANT RESULTS

--------------------------------------------------------------------------------
1. PHASE 1 EXPERIMENT: 3-MODEL ABLATION STUDY
--------------------------------------------------------------------------------
Research Question: Does temporal memory (GRU) improve end-to-end driving, and is 
a CNN spatial backbone necessary when using temporal processing?

Models Tested:
- baseline_cnn:  ResNet18 + Speed -> Control (11.4M params)
- cnn_gru:       ResNet18 -> GRU + Speed -> Control (12.3M params)
- gru_only:      Flatten -> Linear -> GRU + Speed -> Control (78.2M params)

Dataset: ~100k frames (Town01/02/04, Clear & Adverse weather)
Strategy: 3 Bagged Estimators per model architecture to reduce variance.

--------------------------------------------------------------------------------
4. CRITICAL INSIGHTS FOR PRESENTATION & REPORT
--------------------------------------------------------------------------------
KEY INSIGHT 1: The "Golden Nugget" (gru_only failure)
- gru_only scored a Steer R² of 0.0012, meaning it entirely failed to learn how 
  to steer. It just predicted the mean steering value every time.
- HYPOTHESIS CONFIRMED: This conclusively proves that architectural inductive bias 
  (convolutions) is necessary for visual spatial tasks like driving. Raw parameter 
  count (78M vs 11M) cannot compensate for the lack of a CNN backbone.

KEY INSIGHT 2: The "Negative Result" Win (baseline vs cnn_gru)
- baseline_cnn (single frame) comprehensively beat the cnn_gru model across all 
  metrics, despite having fewer parameters and no temporal context.
- REASON A - Markovian Task: Driving in our setup is mostly a "present-tense" 
  task. A single image + instantaneous speed contains 99% of the information 
  needed to make the correct control decision.
- REASON B - Causal Confusion / Inertia: The GRU sequence model learned lazy 
  shortcuts ("inertia"). It outputted whatever it outputted a fraction of a 
  second ago rather than robustly mapping visual features to actions.
- REASON C - Data Starvation: 100k frames equals 100k independent datapoints for 
  the CNN, but only 20,000 independent sequences of length 5 for the GRU, making 
  optimization harder.

--------------------------------------------------------------------------------
5. NEXT ACTIONS & DECISIONS
--------------------------------------------------------------------------------
- Winner: baseline_cnn
- Next Phase (Phase 2): Drop GRU completely. Use baseline_cnn as the foundation 
  and extend it to be "Path-Aware" (adding high-level navigational commands from 
  the CARLA route planner into the model).
